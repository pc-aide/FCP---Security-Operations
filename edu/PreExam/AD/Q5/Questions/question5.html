Refer to the exhibits.

An administrator creates a new address object on the root FortiGate (Local-FortiGate) in the security fabric. After synchronization, this object is not available on the downstream FortiGate (ISFW).

What must the administrator do to synchronize the address object?





Explanation:

When both devices are configured with set downstream-access-disable (answer in A), the newly created address objects are still replicated. However, when the root is configured with set fabric-object-unification local, the address object is no longer replicated to downstream FortiGates.

The CLI command set fabric-object-unification is only available on the root FortiGate. When set to local, global objects will not be synchronized to downstream devices in the Security Fabric. The default value is default.

References:

  • Fortinet Documentation - Synchronizing Objects Across the Security Fabric
  • Fortinet FortiGate Security Study Guide for FortiOS 7.2

Correct Answer:
A. Change the csf setting on Local-FortiGate (root) to set fabric-object-unification default.
