Question 16

An administrator wants to configure dead peer detection (DPD) on IPsec VPN for detecting dead tunnels. The requirement is that FortiGate sends DPD probes only when there is outbound traffic but no response from the peer.

Which DPD mode on FortiGate meets this requirement?





Explanation:

On Demand = Probing when there's outbound traffic but no response.

On Idle = Probing when there's no interesting traffic at all.

In FortiGate, On Demand mode for Dead Peer Detection (DPD) sends DPD probes only when there is outbound traffic but no response from the peer. This mode is efficient for detecting dead tunnels without constantly probing, as it only initiates probes when necessary, meeting the specified requirement.

References:

  • Technical Tip: Configuring DPD on IPsec VPN
  • FortiGate IPsec Phase 1 Parameters

Correct Answer:
A. On Demand
